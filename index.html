Frequently Asked Questions

1. The difference between var, let, and const

var: Function-scoped, can be re-declared and updated, hoisted but initialized as undefined.
let: Block-scoped, can be updated but not re-declared within the same scope, also hoisted but not initialized.
const: Block-scoped, cannot be updated or re-declared, must be initialized at declaration.

2. The difference between map(), forEach(), and filter()

map(): Returns a new array of transformed elements.
forEach(): Iterates over array elements, but does not return anything.
filter(): Returns a new array of elements that pass a test (predicate).

3. Explain arrow functions and how they are different from regular functions

Arrow functions are a concise syntax for writing functions in JavaScript. They do not bind their own this, arguments, or super. Instead, they capture the this value of the enclosing context. They are not suitable as methods on objects and cannot be used as constructors.

4. How JavaScript Promises work

A Promise in JavaScript is an object that represents the eventual completion or failure of an asynchronous operation. It has three states: pending, fulfilled, or rejected. Promises allow you to chain asynchronous actions with then() and handle errors with catch().

5. How closures work in JavaScript

A closure is formed when an inner function captures variables from its outer function's scope. Even after the outer function has returned, the inner function retains access to those variables, allowing for private data and function factories.
